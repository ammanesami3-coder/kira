## 📄 المرحلة 4 — توليد PDF + بوابة الواتساب المجانية (Baileys) والربط

```
اقرأ CLAUDE.md كاملا. أنت تبني المرحلة 4 من مشروع "كراء". المراحل 0-3 جاهزة (يوجد حجز يُنشأ بحالة pending). لا تكسر السابق.

الهدف: عند كل حجز جديد، توليد PDF احترافي + إرساله تلقائيا لواتساب صاحب الوكالة عبر **بوابة Baileys مجانية مستضافة ذاتيا** (لأن Vercel لا يشغّل اتصالا دائما WebSocket). تتكون هذه المرحلة من 3 أجزاء — يُفضّل تنفيذ الجزء (ب) في repo/جلسة منفصلة.

═══ الجزء (أ): توليد الـ PDF (داخل مشروع Next.js على Vercel) ═══
1. قالب PDF بـ @react-pdf/renderer في components/pdf/: ترويسة بشعار الوكالة، رقم المرجع، معلومات الزبون، تفاصيل السيارة، التواريخ ومكان الاستلام/الإرجاع، الإضافات، المجموع، تذييل.
   - **مهم:** سجّل خطا عربيا (Font.register) في @react-pdf/renderer ليُعرض النص العربي RTL بشكل صحيح داخل الـ PDF.
2. lib/pdf.ts: دالة تولّد الـ PDF كـ Buffer من بيانات الحجز.
3. وسّع Server Action للحجز (من المرحلة 3): بعد إنشاء الحجز، ولّد الـ PDF، ارفعه إلى Supabase Storage، احصل على رابط، وخزّنه في bookings.pdf_url.

═══ الجزء (ب): بوابة الواتساب المنفصلة kira-wa-gateway (لا تعيش على Vercel) ═══
أنشئ خدمة Node.js صغيرة مستقلة (repo منفصل) تعمل على مضيف دائم التشغيل (Oracle Cloud Always Free / Raspberry Pi / VPS رخيص):
4. خادم Express + مكتبة baileys: makeWASocket مع useMultiFileAuthState('auth_info_baileys') (جلسة محفوظة على القرص) + fetchLatestBaileysVersion + إعادة اتصال تلقائية عند الانقطاع (تعامل مع DisconnectReason.loggedOut).
5. Endpoints:
   - GET /qr → يعرض/يرجع رمز QR للمصادقة أول مرة.
   - GET /status → حالة الاتصال (connected/disconnected).
   - POST /send-document → محمي بـ header فيه API_KEY؛ يستقبل { to, fileUrl أو base64, filename, caption }؛ يرسل المستند (PDF) عبر sock.sendMessage بنوع document.
6. Dockerfile + .env (API_KEY, PORT) + README مختصر للتشغيل على Oracle Cloud (systemd أو Docker) ومسح QR.
> بديل بلا كتابة كود لهذا الجزء: شغّل حاوية **Evolution API** (Docker) واستعمل REST endpoints ديالها لإرسال المستند بدل بناء خادم Baileys يدويا. الباقي (الجزء ج) يبقى كما هو.

═══ الجزء (ج): الربط من Vercel ═══
7. lib/wa-gateway.ts: عميل ينادي `WA_GATEWAY_URL/send-document` مع WA_GATEWAY_API_KEY في الـ header، يرسل { to: AGENCY_WHATSAPP_NUMBER, fileUrl: bookings.pdf_url, filename, caption يحتوي ملخص الحجز ورقم المرجع }.
8. بعد رفع الـ PDF (الجزء أ): نادِ الـ gateway لإرسال المستند؛ عيّن whatsapp_sent = true عند النجاح.
9. المرونة: إذا كانت البوابة غير متاحة (down/disconnected)، **لا تُفشل الحجز** — سجّل الحالة كـ "بانتظار الإرسال" وأعد المحاولة (retry/backoff)، مع fallback اختياري (إيميل لصاحب الوكالة عبر Resend، أو إظهار رابط wa.me في اللوحة).
10. idempotency: لا ترسل PDF مرتين لنفس الحجز (تحقق من whatsapp_sent قبل الإرسال).

قيود:
- استعمل @react-pdf/renderer (وليس Puppeteer/Playwright — غير متوافق مع Vercel serverless).
- **لا تحاول تشغيل Baileys داخل Next.js/Vercel** — يجب أن يكون في الخدمة المنفصلة فقط.
- الأسرار (WA_GATEWAY_URL, WA_GATEWAY_API_KEY) server-only في Next.js؛ و API_KEY داخل البوابة فقط.
- الإرسال من السيرفر فقط، أبدا من المتصفح.

Definition of Done:
☐ حجز جديد يولّد PDF صحيحا والنص العربي يُعرض بشكل سليم.
☐ بوابة kira-wa-gateway تتصل بالواتساب (مسح QR ناجح)، تبقى متصلة، وتعيد الاتصال تلقائيا.
☐ POST /send-document محمي بـ API key ويرسل الـ PDF فعلا لواتساب صاحب الوكالة (اختبر بإرسال حقيقي).
☐ بعد الحجز: pdf_url محفوظ، نداء البوابة ناجح، whatsapp_sent = true.
☐ عند تعطّل البوابة: الحجز لا يفشل، والإرسال يُعاد لاحقا؛ لا إرسال مكرر.
☐ الأسرار غير مكشوفة في bundle العميل.
☐ pnpm build و lint يمران (لمشروع Next.js).

في النهاية: اعرض نموذج الـ PDF، تأكيد وصول الرسالة عبر البوابة، ونتائج DoD.
```
