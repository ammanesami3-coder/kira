01

الملخص التنفيذي

كراء نظام ويب متكامل لوكالة كراء سيارات واحدة، يُبنى بـ Next.js 16 على Vercel مع Supabase كقاعدة بيانات ومصادقة وتخزين. الموقع العمومي يعرض الأسطول بتصميم فاخر وسريع ويستقبل حجوزات الضيوف بدون تسجيل؛ وعند كل حجز يُولَّد PDF احترافي ويُرسَل تلقائيا لواتساب صاحب الوكالة عبر بوابة Baileys مجانية مستضافة ذاتيا. لوحة التحكم تمكّن صاحب الوكالة من إدارة السيارات والتوفر والحجوزات والهوية بالكامل. كل شيء مبني حول أربع أولويات: الأداء، التجاوب، التصميم، و SEO — وبتكلفة تشغيل تقترب من الصفر، مع بنية قابلة لإعادة النشر لأي وكالة بضبط الإعدادات فقط.

يصل الـ PDF كمستند تلقائيا
لماذا بوابة منفصلة؟ مكتبات مثل Baileys و whatsapp-web.js تحتاج اتصال WebSocket دائم وجلسة محفوظة على القرص — وهذا لا يتوفر في دوال Vercel serverless القصيرة العمر. لذا تعمل البوابة كخدمة صغيرة مستقلة على جهاز دائم التشغيل، و Next.js يناديها عبر webhook بعد كل حجز.
Oracle Cloud Always Free · مجاني دائم
VM كامل (24GB RAM / 4 OCPU) — الأقوى مجانا. يتطلب كرت للتحقق فقط بلا خصم.
Raspberry Pi / حاسوب المكتب · بلا كرت
مثالي لوكالة لها مكتب فعلي — جهاز دائم التشغيل + كهرباء فقط.
VPS رخيص (~$4) · اختياري
إذا أردت استضافة مُدارة وموثوقة دون إدارة جهاز فعلي.
مقايضة المجانية: Baileys غير رسمي (يحاكي WhatsApp Web)؛ خطر حظر الرقم منخفض جدا لإشعار حجز واحد، ويُفضّل رقم مخصص. عند انقطاع الجلسة أو تعطّل البوابة، لا يفشل الحجز — يُسجَّل للإرسال ويُعاد تلقائيا (retry)، مع fallback اختياري (إيميل عبر Resend أو رابط wa.me في اللوحة).

Core Web Vitals

  • LCP < 2.5s · CLS < 0.1 · INP < 200ms
  • next/image (AVIF/WebP) + priority لصورة LCP
  • خطوط display:swap + subsetting
  • ISR/caching + تقليل JS على العميل
🌍

التعدد اللغوي والفهرسة
